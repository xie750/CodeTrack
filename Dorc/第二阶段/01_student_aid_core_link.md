# 助学核心链路

## 1. 总体链路

第二阶段学生端围绕两条学习路径展开：

```text
路径 A：教师任务驱动
学习首页 -> 课程任务 -> 任务工作台 -> 测试与诊断 -> 分层提示
-> 再次提交 -> 学习总结 -> 画像更新 -> 资料沉淀

路径 B：学生自主学习
学习首页 -> 自主学习 -> 选择知识点 -> AI 讲解/练习/资料生成
-> 保存资料 -> 画像更新 -> 推荐下一步
```

两条路径共享同一个学习画像、课程知识库和资料库。

## 2. 学习首页链路

学生进入系统后先看到：

- 今日待完成任务；
- 当前学习进度；
- 近期薄弱知识点；
- 系统推荐的下一步；
- 最近保存的学习资料；
- AI 助学快捷入口。

学习首页不承担复杂操作，只负责让学生知道“现在该做什么”。

## 3. 教师任务链路

### 3.1 进入任务

学生从任务列表进入教师分发的作业或测试。任务需要展示：

- 课程名称；
- 任务名称；
- 截止时间；
- 任务状态；
- 涉及知识点；
- 难度；
- 最近一次提交结果。

### 3.2 任务工作台

任务工作台保留第一阶段核心体验，但调整信息结构：

- 左侧：任务说明、知识点、评价要求、公开样例；
- 中间：代码编辑器和提交操作；
- 右侧：系统验证、AI 诊断、分层提示、知识来源。

移动端可改为分段 Tab，不强行保持三栏。

### 3.3 测试与诊断

系统先展示工具事实，再展示 AI 解释：

1. 编译状态；
2. 测试通过情况；
3. 失败用例摘要；
4. 错因类型；
5. AI 解释；
6. 引用的课程知识；
7. 下一步建议。

AI 诊断不能覆盖工具验证结果，必须明确标注“AI 生成内容”。

### 3.4 分层提示

提示分三层：

- 一级提示：指出方向和相关知识点，不定位具体代码。
- 二级提示：指出可能出错的逻辑分支或边界条件。
- 三级提示：给出修复思路或局部伪代码，但不直接贴完整答案。

如果题目处于考核模式，三级提示需要显示风险提示或由教师配置是否开放。

### 3.5 学习总结

任务通过后生成总结：

- 本次任务掌握情况；
- 主要错因；
- 修改过程；
- 使用过的提示；
- 关联知识点；
- 推荐复习资料；
- 可保存的学习笔记。

总结自动进入“我的资料”，同时更新学习画像。

## 4. 自主学习链路

### 4.1 选择知识点

学生可以从知识地图选择：

- 链表；
- 栈与队列；
- 树；
- 图；
- 查找；
- 排序；
- 复杂度分析。

第一版可以只开放链表、栈与队列、树三个主题。

### 4.2 AI 生成学习内容

学生可以选择生成：

- 概念讲解；
- 分步例题；
- 练习题；
- 复习笔记；
- 知识卡片；
- 思维导图；
- PPT 大纲。

第一版不生成真实视频，只保留“视频讲解脚本”作为后续扩展点。

### 4.3 保存资料

AI 输出支持保存到资料库。保存时需要记录：

- 资料类型；
- 关联知识点；
- 生成来源；
- 引用知识源；
- 学生备注；
- 创建时间。

## 5. AI 助学链路

AI 助学不是独立聊天工具，而是贯穿任务和自主学习：

```text
用户问题
-> 读取当前页面上下文
-> 读取学生画像
-> 检索课程知识库
-> 生成回答
-> 标注引用来源
-> 给出下一步动作
```

下一步动作包括：

- 继续追问；
- 生成练习；
- 生成笔记；
- 保存到资料；
- 跳转到相关任务；
- 更新学习计划。
