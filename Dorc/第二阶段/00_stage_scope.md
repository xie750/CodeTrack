# 第二阶段范围说明

## 1. 产品定位

CodeTrack 第二阶段定位为：

> 面向计算机类专业基础课程的个性化助学系统。

重点课程方向先定为“数据结构与程序设计基础”。该方向可以自然覆盖代码调试、概念辨析、算法理解、作业练习和学习路径规划，符合赛题中“学科垂类大模型与创新应用”的要求。

## 2. 本阶段只做助学

本阶段只规划学生端助学场景，暂不规划：

- 助教场景；
- 助研场景；
- 完整教务系统；
- 多学科通用平台；
- 复杂组织、班级和权限后台。

教师分发任务只作为学生端输入来源出现，不展开教师端完整工作流。

## 3. 保留的核心链路

第一阶段中可复用的核心能力只保留为底层链路：

```text
教师分发任务
-> 学生进入任务
-> 学生编辑和提交代码
-> 系统运行测试
-> AI 基于测试结果和课程知识库诊断
-> 学生按层级获取提示
-> 学生修改并再次提交
-> 通过后生成学习总结
-> 更新学习画像
-> 推荐下一步学习内容
-> 保存学习资料
```

## 4. 新增的助学能力

第二阶段新增四类助学能力：

1. 学习画像：根据任务、错因、提示使用、学习资料生成记录形成学生状态。
2. 自主学习：学生可以选择知识点，自主生成学习路径、讲解、练习和资料。
3. AI 助学：AI 回答需要结合课程知识库、当前页面上下文和学生画像。
4. 资料沉淀：AI 输出可以保存为笔记、知识卡片、错题总结、思维导图或 PPT 大纲。

## 5. 关键取舍

### 本阶段必须做

- 学生端导航和页面骨架；
- 学习首页；
- 教师任务列表；
- 编程任务工作台；
- AI 答疑入口；
- 自主学习入口；
- 我的资料库；
- 学习画像摘要；
- 可追溯的课程知识引用。

### 本阶段暂缓

- 真实视频生成；
- 完整 PPT 文件生成；
- 多模态文件深度解析；
- 完整教师端看板；
- 助研文献追踪；
- 大规模用户运营功能；
- 积分、勋章、排行榜等弱相关激励。

## 6. 成功标准

第二阶段最小可用版本需要让评审能看到：

- 它不是普通判题系统，而是专业课助学系统；
- 它不是通用聊天机器人，而是结合课程知识库和学生画像的垂类 AI；
- 它能完整展示学生从任务、诊断、提示、复习到资料沉淀的学习闭环；
- 它的功能边界清晰，可继续扩展到助教和助研。
