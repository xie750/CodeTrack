# 第一阶段产品决策基线

本文档用于把首个 Demo 编码前必须确认的产品决策收束为可执行基线。除“部署环境”外，以下决策在首个 Demo 开发期间按已冻结处理；如需修改，必须同步更新范围、API、数据、测试和验收文档。

## 1. 项目命名

**状态：已冻结**

- 对外产品名：CodeTrack。
- 仓库、目录、页面、演示脚本和产品文案统一使用 `CodeTrack`。
- 不再使用其他中英文产品名，避免在用户界面和文档中混用多个品牌名。

## 2. 技术栈

**状态：已冻结为第一阶段基线**

- 前端：React + TypeScript + Vite。
- UI：成熟组件库，不自研基础组件。
- 后端：FastAPI + SQLAlchemy + Alembic。
- 数据库：PostgreSQL，向量检索优先使用 pgvector。
- 异步任务：Redis + Worker。
- 沙箱：独立 Docker 容器执行服务，业务后端不得直接运行学生代码。
- 接口文档：OpenAPI。
- 测试：pytest + 前端组件测试或端到端测试。

说明：首个 Demo 以真实闭环为目标，不为未来大规模扩展提前引入复杂微服务。

## 3. 大模型接入

**状态：产品合同已冻结，具体供应商待外部约束确认**

第一阶段不在业务代码中绑定具体模型供应商，统一通过模型网关或适配层调用。

必须满足：

- 支持配置主模型；
- 预留备用模型或模板化降级出口；
- 模型输出必须通过 JSON Schema 校验；
- 诊断必须引用真实测试证据和真实知识来源；
- 模型失败不得影响编译结果、测试结果和重新提交；
- 不允许前端直接访问模型供应商；
- 不允许用写死诊断冒充模型能力。

仍需在接入前确认：

- 是否受赛事指定模型约束；
- 主模型和备用模型名称；
- 预算、限流和数据条款。

## 4. 代码编辑方式

**状态：已冻结**

首个 Demo 使用 Monaco 或成熟代码编辑器组件。

要求：

- 支持 C++ 语法高亮；
- 支持复制粘贴预置错误代码；
- 支持提交前基础校验；
- 不自研编辑器核心能力；
- 编辑器只负责输入，不承载判题、诊断或提示逻辑。

## 5. 任务协议

**状态：已冻结**

首个任务采用固定函数签名，不采用完整标准输入输出程序。

固定接口：

```cpp
ListNode* deleteAt(ListNode* head, int position);
```

协议规则：

- 平台提供 `ListNode` 结构、测试驱动和编译模板；
- 学生只实现指定函数，不编写 `main`；
- 沙箱使用受控测试驱动编译学生代码；
- 前端展示函数签名、参数含义、返回值要求和公开测试说明；
- 隐藏测试不暴露完整输入和判定逻辑，只展示失败摘要；
- 标准错误代码必须在“删除头节点”测试中失败；
- 正确代码必须通过全部必要测试。

公开测试首版包含：

- 删除中间节点；
- 删除头节点；
- 空链表删除。

可选隐藏或扩展测试包含：

- 删除尾节点；
- 非法位置。

## 6. 课程资料

**状态：已冻结为最小资料集，具体文本需补齐**

首个 Demo 的 RAG 知识库只收录与单链表删除相关的少量审核资料。

必须准备：

- 实验任务说明；
- 单链表删除规则说明；
- 头节点删除边界说明；
- 空链表处理说明；
- 教师参考提示；
- 标准错误案例；
- 资料来源、版本和权威等级。

资料要求：

- 每条知识必须有 `source_id`；
- AI 诊断不得伪造不存在的来源；
- 引用内容必须能回溯到原始条目；
- 未准备授权材料时，先使用教师自编或可授权摘要，不直接复制受版权保护的大段教材内容。

## 7. 登录与账号

**状态：已冻结**

首个 Demo 使用预置测试账号 + 简单登录。

账号范围：

- 一个教师账号；
- 一个或两个学生账号；
- 可选一个管理员或配置账号，仅用于内部初始化。

要求：

- 后端仍必须校验角色、课程成员关系和资源归属；
- 不能只通过前端隐藏按钮实现权限；
- 登录流程不作为第一阶段主要产品亮点。

## 8. 部署环境

**状态：延期**

部署环境不是当前阶段工作重点，暂不冻结服务器、域名、HTTPS 和正式发布方案。

但本地开发仍需保留：

- 环境变量样例；
- Docker 或等价沙箱运行说明；
- 健康检查接口；
- 不提交真实密钥。

部署相关决策在 Demo 主链路完成后单独评审。

## 9. 能力状态命名

**状态：已冻结**

第一阶段能力状态使用：

- 待观察；
- 需要支持；
- 初步掌握；
- 基本掌握。

不展示百分制能力分，不把单次模型判断作为最终能力等级。

## 10. 教师端范围

**状态：已冻结**

首个 Demo 教师端只保证：

- 学生提交列表；
- 单个提交时间线；
- 版本、执行、测试、诊断、提示和能力证据事件；
- 错因标签；
- 最高提示层级；
- 最终是否通过。

班级统计和教学建议可以使用少量真实操作数据或预置样例数据，但必须标注数据来源。复杂统计大屏不进入首个 Demo 验收范围。

## 11. 下一次评审重点

正式编码前只需评审以下内容：

1. 固定函数任务协议与测试驱动；
2. 学生任务工作台布局；
3. 数据模型与 API 合同；
4. 沙箱协议和资源限制；
5. AI 输出 Schema 与泄露检查；
6. 验收测试用例。
