编程任务：两数之和

难度：中等

题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出两个整数，使它们的和与目标值 target 相等。

你可以假设每个输入只对应一种答案，并且你不可以重复使用相同的元素。

输入输出说明
  • 输入：整数数组 nums 和整数 target
  • 输出：返回这两个整数在数组中的下标，顺序不限。
示例

输入：nums = [2,7,11,15], target = 9

输出：[0,1]

解释：因为 nums[0] + nums[1] == 9，所以返回 [0,1]

约束条件
  • 2 <= nums.length <= 10^4
  • -10^9 <= nums[i] <= 10^9
  • -10^9 <= target <= 10^9
  • 只会存在一个有效答案
知识点
哈希表数组两数之和时间复杂度
老师备注

请尽量使用 O(n) 时间复杂度和 O(n) 空间复杂度完成。

可使用哈希表存储遍历过程中的数值与下标。

代码编辑器

1class Solution {
2public:
3 vector<int> twoSum(vector<int>& nums, int target) {
4 unordered_map<int, int> mp;
5 for (int i = 0; i < nums.size(); i++) {
6 int complement = target - nums[i];
7 if (mp.count(complement)) {
8 return {mp[complement], i};
9 }
10 mp[nums[i]] = i;
11 }
12 return {};
13 }
14};

AI诊断总结

你的代码整体思路正确，使用哈希表查找补数，时间复杂度为 O(n)。但在处理重复元素或存储顺序上仍存在问题，导致在某些情况下未能找到有效解。

问题分析

  • 失败测试点：测试点 5（[0,4,3,0], target = 0）
  • 原因：在处理当前元素时，未先查询补数，可能将当前元素的下标提前存入哈希表。

分层提示

第一层提示 · 思考方向 ⌄
思考在遍历数组时，如何正确处理相同的元素值？关注“查询补数”和“存入当前元素下标”的时机。
第二层提示 · 逻辑分析 ⌄
对于每个元素 nums[i]：
1. 先在哈希表中查找 target - nums[i] 是否存在；
2. 如果存在，直接返回对应下标；
3. 如果不存在，再把当前元素及其下标存入哈希表。
第三层提示 · 关键步骤 / 伪代码 ⌄
1. 创建哈希表 mp，用于存储“数值 → 下标”；
2. 遍历 nums，计算 complement；
3. 若 complement 已存在则返回，否则存入当前元素；
4. 遍历结束未找到则返回空结果。
